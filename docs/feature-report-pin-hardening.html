Alur Rate Limiting

  • Client kirim auth → Server panggil check_rate_limit(ip)
  • Jika diblokir: server balas {"t":"auth_fail","reason":"rate_limit"}
  • Jika diizinkan: verifikasi PIN →
    • Benar → reset_failed_attempts(ip) → auth_ok
    • Salah → record_failed_attempt(ip) → {"t":"auth_fail","reason":"pin"}
  • HP menampilkan pesan yang sesuai berdasarkan reason

Catatan

  • Tidak ada breaking change — server lama menerima PIN 4 digit dari APK lama
  • APK baru tetap bisa konek ke server lama (PIN dikirim apa adanya)
  • Pengguna perlu mengetik ulang PIN baru di HP setelah update server
  • Token pairing yang sudah ada tetap berlaku — tidak perlu pairing ulang
