RSA-2048 Encrypted Handshake

PASSED

Masalah

PIN dikirim dalam plaintext saat handshake. Penyadap WiFi dapat menangkapnya dan menyamar sebagai HP yang sah.

Solusi

AspekSebelumSesudah
Transmisi PINPlaintext JSONRSA-2048 OAEP (SHA-256) terenkripsi
Key exchangeTidak adaServer generate keypair, kirim publik ke HP
Backward compat—Fallback ke plaintext untuk server lama

Alur Handshake

  • HP kirim hello
  • Server balas hello_ok + salt + pubkey (base64)
  • HP enkripsi PIN/token dengan RSA public key → kirim pin_enc / token_enc
  • Server dekripsi dengan private key, verifikasi PIN

Binary Protocol

v3.3 PASSED

Masalah

Semua command dikirim sebagai JSON. {"t":"move","dx":5,"dy":-3} = 30 bytes. Di WiFi dengan latency tinggi, overhead ini menambah delay terasa.

Solusi

AspekSebelum (JSON)Sesudah (Binary)
Ukuran move~30 bytes5 bytes (83% lebih kecil)
Ukuran click~25 bytes3 bytes (88% lebih kecil)
Ukuran scroll~28 bytes5 bytes (82% lebih kecil)
Negotiation—Otomatis via auth handshake
Backward compat—Fallback ke JSON untuk server lama

Wire Format

Setiap paket: [CMD_ID: 1 byte][PAYLOAD: 0..N bytes], little-endian. 16 command IDs (0x01–0x10):

IDCommandPayloadUkuran
0x01movedx: i16, dy: i165 bytes
0x02clickbutton_id: u8, double: u83 bytes
0x05scrolldy: i16, dx: i165 bytes
0x06zoomdir: i82 bytes
0x08textlen: u16 + UTF-8 bytesvariable
0x09keyvk: u8 + mods: [u8]variable
0x10ping(empty)1 byte
